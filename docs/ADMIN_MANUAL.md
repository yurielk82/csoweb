## 1. 관리자 대시보드

로그인 후 첫 화면입니다. 시스템 전체 상태를 한눈에 파악할 수 있습니다.

### 1.1 통계 카드

| 카드 | 내용 | 조치 |
|------|------|------|
| 승인 대기 | 아직 승인하지 않은 가입 신청 건수 | 클릭 → 회원 승인 페이지 |
| 전체 업체 | 등록된 전체 업체 수 | |
| 이메일 발송 | 최근 성공/실패 건수 | |

### 1.2 빠른 작업

7개의 바로가기 버튼이 제공됩니다.

| 버튼 | 기능 | 비고 |
|------|------|------|
| 정산서 업로드 | 엑셀 파일 업로드 | |
| 거래처 매핑 | CSO ↔ 업체 매칭 관리 | |
| 회원 승인 | 가입 신청 처리 | 대기 건수 배지 표시 |
| 데이터 관리 | 정산월별 데이터 관리 | |
| 컬럼 설정 | 표시 컬럼 커스터마이징 | |
| 메일머지 | 일괄 이메일 발송 | |
| 이메일 이력 | 발송 로그 조회 | |

### 1.3 시스템 정보

하단에 2열 그리드로 시스템 상태가 표시됩니다. 각 항목에 컬러 아이콘이 표시됩니다.

| 항목 | 아이콘 | 정상 표시 |
|------|--------|-----------|
| 버전 | Tag | 현재 앱 버전 |
| 데이터베이스 | Database | Supabase 연결됨 |
| 국세청 API | FileText | NTS 설정됨 |
| 심평원 병원정보 API | Building2 | HIRA 설정됨 |
| 심평원 약국정보 API | Pill | HIRA 설정됨 |
| 환경 | Globe | Production / Development |
| 이메일 서비스 | Mail | 활성 프로바이더 + 보조 프로바이더 표시 |

> 상태가 **미연결/미설정**이면 해당 서비스 설정을 확인하세요.

## 2. 회원 승인

CSO 업체가 회원가입을 신청하면 관리자가 승인/거부합니다.

### 2.1 승인 처리

1. **회원 승인** 메뉴로 이동합니다.
2. 대기 중인 신청 목록이 카드 형식으로 표시됩니다.
3. 각 신청 건의 **사업자번호, 업체명, 이메일, 신청일**을 확인합니다.
4. **"승인"** 버튼을 클릭합니다.

> 승인 시 해당 업체에 승인 완료 이메일이 자동 발송됩니다.

### 2.2 일괄 승인

1. 각 카드의 체크박스를 선택하거나 **전체 선택**을 클릭합니다.
2. 상단의 **"일괄 승인 (n건)"** 버튼을 클릭합니다.
3. 처리 결과(성공/실패 건수)가 표시됩니다.

### 2.3 거부 처리

1. 거부할 신청의 **"거부"** 버튼을 클릭합니다.
2. 거부 사유를 입력합니다 (선택사항).
3. **"거부"** 를 확인합니다.

> 거부 시 해당 업체에 거부 안내 이메일이 자동 발송됩니다 (사유 포함).

## 3. 정산서 업로드

SIT솔루션에서 다운로드한 엑셀 파일을 업로드합니다.

### 3.1 업로드 절차

1. **정산서 업로드** 메뉴로 이동합니다.
2. 엑셀 파일(.xlsx, .xls)을 **드래그&드롭** 하거나 클릭하여 선택합니다.
   - 파일 크기 제한: **20MB**
3. **"컬럼 매핑 확인"** 버튼을 클릭합니다.

### 3.2 컬럼 매핑

미리보기 다이얼로그가 표시됩니다.

| 항목 | 설명 |
|------|------|
| 자동 매칭율 | 엑셀 컬럼과 DB 컬럼의 자동 매칭 비율 (%) |
| 샘플 데이터 | 처음 3~5행 미리보기 |
| 필수 컬럼 경고 | 사업자번호, 정산월이 매핑되지 않으면 경고 |

4. 매핑을 확인하고 **"업로드 진행"** 버튼을 클릭합니다.
5. 업로드 완료 후 결과(행 건수, 정산월)가 표시됩니다.

### 3.3 주의사항

- **같은 정산월**의 데이터를 다시 업로드하면 **기존 데이터가 자동으로 교체**됩니다.
- 업로드 전 엑셀 파일의 컬럼명이 시스템과 일치하는지 확인하세요.
- 사업자번호 컬럼이 반드시 포함되어야 합니다.

## 4. 마스터 데이터 조회

전체 CSO 업체의 정산 데이터를 통합 조회합니다. 일반 사용자 화면과 달리 **모든 업체**의 데이터를 볼 수 있습니다.

### 4.1 조회 조건

| 조건 | 설명 |
|------|------|
| 정산월 | 드롭다운에서 선택 |
| 거래처 검색 | 회사명 또는 사업자번호로 특정 업체 필터링 |
| 제품/거래처 검색 | 자유 텍스트 검색 |
| 표시 컬럼 | 체크박스로 선택 |

### 4.2 요약 통계

3개 카드: **총 수량**, **총 금액**, **제약수수료 합계**

### 4.3 Notice 편집

정산서 조회 화면에 표시되는 안내사항을 편집합니다.

1. Notice 영역의 **"편집"** 버튼을 클릭합니다.
2. 내용을 수정합니다.
3. **"저장"** 버튼을 클릭합니다.

**사용 가능한 변수:**

| 변수 | 치환 결과 | 예시 |
|------|-----------|------|
| `{{정산월}}` | 사용자가 조회 중인 정산월 | 1월 |
| `{{정산월+1}}` | 다음 달 | 2월 |
| `{{대표자명}}` | 사이트 설정의 대표자명 | 홍길동 |

### 4.4 엑셀 다운로드

선택한 표시 항목 기준으로 전체 데이터를 엑셀로 다운로드합니다.

## 5. 메일머지

CSO 업체에 정산 안내 이메일을 개인화하여 일괄 발송합니다.

### 5.1 수신 대상 선택

| 옵션 | 설명 |
|------|------|
| 전체 승인된 업체 | 승인 상태인 모든 업체에 발송 |
| 특정 정산월 | 해당 월에 정산 데이터가 있는 업체만 발송 |

> 수신자 수가 배지로 실시간 표시됩니다.

### 5.2 메일 작성

**제목**과 **본문**을 작성합니다. 아래 변수를 사용하면 각 업체별로 자동 치환됩니다.

| 변수 | 내용 | 예시 |
|------|------|------|
| `{{업체명}}` | 업체 회사명 | ABC 상사 |
| `{{사업자번호}}` | 사업자번호 | 123-45-67890 |
| `{{이메일}}` | 업체 이메일 | sample@example.com |
| `{{대표자명}}` | 회사 대표자 이름 | 정근호 |
| `{{정산월}}` | 정산 년월 | 2026년 01월 |
| `{{정산월+1}}` | 정산 다음월 | 2월 |
| `{{총_금액}}` | 해당 업체의 총 금액 | 15,234,000원 |
| `{{총_수수료}}` | 해당 업체의 총 수수료 (= 세금계산서 발행 금액) | 1,781,729원 |
| `{{제약수수료_합계}}` | 제약수수료 합계 | 1,781,729원 |
| `{{담당수수료_합계}}` | 담당수수료 합계 | 523,400원 |
| `{{총_수량}}` | 총 수량 | 1,250 |
| `{{데이터_건수}}` | 데이터 건수 | 127 |

> 변수 배지를 클릭하면 커서 위치에 자동 삽입됩니다.
> Notice 본문에도 변수를 사용할 수 있으며, 발송 시 각 업체별로 자동 치환됩니다.

### 5.3 미리보기

**"미리보기"** 버튼을 클릭하면 첫 번째 수신자 기준으로 변수가 치환된 결과를 확인할 수 있습니다.

### 5.4 발송

1. **"발송"** 버튼을 클릭합니다.
2. 실시간 진행 상황이 표시됩니다.
   - **프로그레스 바** + 퍼센트
   - 성공/실패 건수
   - 남은 시간 예측
   - 각 업체별 발송 결과 로그 (실시간 스크롤)
3. 발송 중 **"중단"** 버튼으로 중지할 수 있습니다.

### 5.5 주의사항

- 스팸 필터 방지를 위해 각 이메일 사이에 **지연 시간**(기본 6초)이 적용됩니다.
- 지연 시간은 **사이트 설정 → 일괄 발송 지연**에서 조정할 수 있습니다.
- 메일머지는 이메일 알림 ON/OFF 설정의 영향을 받지 않습니다 (항상 발송).

## 6. 회원 관리

등록된 전체 회원을 조회하고 관리합니다.

### 6.1 회원 목록

상단 통계 카드(전체, 관리자, 승인됨, 대기중)를 클릭하면 해당 상태로 필터링됩니다.

### 6.2 회원 정보 수정

1. 해당 회원의 **"수정"** 버튼을 클릭합니다.
2. 수정 다이얼로그에서 정보를 변경합니다.
   - 업체명, 대표자명
   - 주소 (우편번호, 도로명, 상세)
   - 연락처, 이메일
   - 관리자 권한 토글
   - 승인 상태 토글
3. **"저장"** 버튼을 클릭합니다.

### 6.3 비밀번호 초기화

사용자가 비밀번호를 분실한 경우 관리자가 초기화할 수 있습니다.

1. 해당 회원의 **"비밀번호"** 버튼을 클릭합니다.
2. 확인 다이얼로그에서 **"초기화"** 를 클릭합니다.
3. 임시 비밀번호가 화면에 표시됩니다: `u{사업자번호숫자}`
   - 예: 사업자번호 `100-12-34567` → 임시 비밀번호 `u1001234567`
4. 이 비밀번호를 해당 업체에 전달합니다.

> 사용자는 임시 비밀번호로 로그인 후 반드시 새 비밀번호를 설정해야 합니다.

### 6.4 회원 삭제

1. 해당 회원의 **"삭제"** 버튼을 클릭합니다.
2. 확인 다이얼로그에서 **"삭제"** 를 클릭합니다.

> **삭제는 되돌릴 수 없습니다.** 해당 회원의 계정과 프로필 정보가 영구 삭제됩니다.

### 6.5 엑셀 다운로드

회원 목록을 엑셀 파일로 다운로드할 수 있습니다.

## 7. 컬럼 설정

사용자 화면(정산서 조회, 월별 합계)에 표시되는 컬럼을 커스터마이징합니다.

### 7.1 설정 항목

각 컬럼에 대해 아래 옵션을 설정할 수 있습니다.

| 옵션 | 설명 |
|------|------|
| 표시 | 사용자 화면에 보이는지 여부 |
| 표시명 | 사용자에게 보이는 컬럼 이름 (수정 가능) |
| 필수 | 사용자가 해제할 수 없는 필수 표시 컬럼 |
| 합계 | 월별 합계 페이지에서 합산 대상 여부 (숫자형 컬럼만) |
| 순서 | 드래그&드롭으로 표시 순서 변경 |

### 7.2 순서 변경

1. 컬럼 좌측의 **그립 핸들(⠿)** 을 클릭한 채로 드래그합니다.
2. 원하는 위치에 놓습니다.
3. **"저장"** 버튼을 클릭합니다.

### 7.3 일괄 작업

| 버튼 | 기능 |
|------|------|
| 전체 표시 | 모든 컬럼을 표시 상태로 변경 |
| 전체 숨김 | 모든 컬럼을 숨김 (필수 제외) |
| 기본값 복원 | 초기 설정으로 되돌림 |

## 8. 정산 데이터 관리

업로드된 정산 데이터를 정산월 단위로 관리합니다.

### 8.1 화면 구성

| 카드 | 내용 |
|------|------|
| 총 데이터 건수 | 전체 업로드된 행 수 |
| 정산월 수 | 업로드된 월 수 |
| 관련 업체 수 | 데이터에 포함된 업체 수 |

### 8.2 정산월별 테이블

각 정산월에 대해 아래 정보가 표시됩니다.

- 정산월, 처방월
- CSO관리업체 수
- 수량/금액/제약수수료 합계

### 8.3 정산월 삭제

1. 삭제할 정산월의 **"삭제"** 버튼을 클릭합니다.
2. 확인 다이얼로그에서 **"삭제"** 를 클릭합니다.

> **주의:** 해당 정산월의 **모든 데이터가 영구 삭제**됩니다. 되돌릴 수 없습니다.

> 같은 정산월의 데이터를 교체하려면 삭제 없이 다시 업로드하면 자동으로 교체됩니다.

## 9. 이메일 발송 이력

시스템에서 발송된 모든 이메일의 로그를 조회합니다.

### 9.1 필터

| 필터 | 옵션 |
|------|------|
| 템플릿 유형 | 가입 신청 알림, 승인 완료, 거부 안내, 정산서 업로드, 비밀번호 재설정, 메일머지 |
| 상태 | 전체, 대기중, 발송됨, 실패 |

### 9.2 로그 정보

| 컬럼 | 내용 |
|------|------|
| 발송일시 | 이메일 발송 시간 |
| 수신자 | 수신 이메일 주소 |
| 제목 | 이메일 제목 |
| 유형 | 이메일 템플릿 유형 (배지) |
| 상태 | 발송 성공/실패/대기중 |

> 발송 실패 건은 에러 메시지를 확인하여 원인을 파악할 수 있습니다.

## 10. 거래처 매핑

CSO 관리 거래처와 실제 등록 업체 간의 매칭 현황을 관리합니다.

### 10.1 주요 기능

| 기능 | 설명 |
|------|------|
| 매핑 현황 조회 | 정산 데이터의 CSO관리업체 ↔ 등록 업체 매칭 상태 |
| 매핑 편집 | 행 단위로 업체명 입력/수정 |
| 상태 필터링 | 가입 완료/미가입 업체 필터 |
| 일괄 저장 | 수정된 매핑 정보 일괄 저장 |

### 10.2 매핑 상태 아이콘

| 아이콘 | 의미 |
|--------|------|
| 녹색 체크 | 매칭 완료 — 등록 업체와 정상 연결 |
| 주황색 경고 | 부분 매칭 — 확인 필요 |
| 빨간색 X | 미매칭 — 등록되지 않은 업체 |

## 11. 사이트 설정

시스템 기본 정보와 이메일 설정을 관리합니다.

### 11.1 기본 정보

로그인 화면 하단 푸터에 표시되는 회사 정보입니다.

| 항목 | 설명 |
|------|------|
| 회사명 | 운영 회사명 |
| 대표자명 | 대표이사 이름 |
| 사업자등록번호 | 운영 회사 사업자번호 |
| 주소 | 회사 주소 |
| 전화번호, 팩스 | 연락처 |
| 이메일 | 대표 이메일 |
| 웹사이트 | 회사 홈페이지 URL |

### 11.2 이메일 발송 설정

이메일 발송에 사용할 프로바이더를 설정합니다.

| 프로바이더 | 설정 필요 항목 | 비고 |
|-----------|---------------|------|
| **Resend API** | 없음 (환경변수로 설정) | 기본값, 안정적 |
| **SMTP** | 호스트, 포트, SSL/TLS, 계정, 비밀번호, 발신자 정보 | 하이웍스 등 |

**SMTP 선택 시:**
1. SMTP 서버 정보를 입력합니다.
2. **"연결 테스트"** 버튼으로 연결 상태를 확인합니다.

> **자동 저장:** 텍스트 필드는 입력 후 다른 필드로 이동하면 자동 저장됩니다. 체크박스/라디오는 변경 즉시 저장됩니다. 헤더에 "저장됨" 표시로 확인할 수 있습니다. "전체 저장" 버튼은 모든 필드를 한 번에 저장하는 fallback입니다.

**일괄 발송 지연:** 메일머지 등 대량 발송 시 각 이메일 사이의 대기 시간입니다.
- 범위: 0.5~30초
- 권장: **6초** (스팸 필터 방지)

### 11.3 이메일 알림 ON/OFF

이메일 유형별로 발송 여부를 제어합니다.

| 유형 | 기본값 | 설명 |
|------|:------:|------|
| 회원가입 신청 알림 | ON | 새 가입 신청 시 관리자에게 알림 |
| 가입 승인 알림 | ON | 승인 시 해당 업체에 안내 |
| 가입 거부 알림 | ON | 거부 시 해당 업체에 안내 (사유 포함) |
| 정산서 업로드 알림 | ON | 정산서 업로드 시 해당 업체에 안내 |
| 비밀번호 재설정 | ON | 비밀번호 재설정 링크 발송 |

> **메일머지(수동 발송)** 는 이 설정의 영향을 받지 않으며 항상 발송됩니다.

> **비밀번호 재설정**을 OFF로 설정하면 사용자가 비밀번호를 분실했을 때 이메일로 재설정할 수 없습니다. 관리자가 직접 초기화해야 합니다.

## 12. 월간 운영 체크리스트

매월 정산 업무 시 아래 순서를 참고하세요.

### 정산 데이터 처리

```
1. SIT솔루션에서 엑셀 파일 다운로드
   ↓
2. 정산서 업로드 — 엑셀 파일 업로드 + 컬럼 매핑 확인
   ↓
3. 마스터 데이터 조회 — 업로드된 데이터 검증
   ↓
4. Notice 편집 — 세금계산서 발행일, 마감일 등 공지 업데이트
   ↓
5. 메일머지 — 업체에 정산 안내 이메일 발송
   ↓
6. 이메일 이력 — 발송 결과 확인 (실패 건 재발송)
```

### 회원 관리

- [ ] 신규 가입 신청 확인 및 승인/거부
- [ ] 비밀번호 초기화 요청 처리 (있는 경우)
- [ ] 회원 정보 변경 요청 처리 (있는 경우)

### 시스템 점검

- [ ] 대시보드에서 시스템 상태 확인 (Supabase, 이메일, API 연결)
- [ ] 이메일 발송 이력에서 실패 건 확인
- [ ] 거래처 매핑 상태 확인 (미매칭 업체 유무)
